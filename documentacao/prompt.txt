# Escopo do MVP

O sistema deve conter apenas:

1. autenticação;
2. dashboard simples;
3. cadastro e gerenciamento de crianças;
4. registro de consultas;
5. acompanhamento de crescimento;
6. checklist de desenvolvimento;
7. assistente científico com RAG;
8. geração e exportação de relatórios.

Não implemente funcionalidades pós-MVP.

---

# Stack técnica

Use:

- Next.js com App Router;
- React;
- TypeScript em modo estrito;
- Tailwind CSS;
- Firebase Authentication;
- Cloud Firestore;
- Firebase Storage;
- Zod para validação;
- React Hook Form;
- Vitest;
- React Testing Library;
- Playwright;
- Vercel para deploy.

Evite dependências desnecessárias.

---

# Regras clínicas e de segurança

1. Não implemente diagnóstico automático.
2. Não implemente prescrição de medicamentos.
3. Não invente marcos do desenvolvimento ou recomendações clínicas.
4. Conteúdos clínicos provisórios devem ser claramente identificados como pendentes de validação profissional.
5. Toda resposta do assistente científico deve apresentar as fontes utilizadas.
6. O assistente deve responder somente com base nos documentos científicos cadastrados.
7. Relatórios gerados por IA devem ser revisados e confirmados pelo profissional.
8. Cada usuário deve acessar somente os próprios pacientes, consultas e relatórios.
9. Não armazene segredos, chaves ou credenciais no código.
10. Não use dados clínicos reais durante o desenvolvimento.
11. Dados clínicos devem usar exclusão lógica.
12. Não registre informações clínicas sensíveis em logs.
